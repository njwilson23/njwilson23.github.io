Grid statistics, resizing, resampling, and profile extraction

Karta simplifies management of in- and out-of-memory digital elevation models, satellite imagery, and land classification maps and performs simple sampling and transformation operations.

Regularly-spaced grids are represented by RegularGrid, with data backed by numpy arrays. Grids with arbitrary spacing can be represented by the experimental WarpedGrid. Grids can be clipped, resampled, queried at geographical points, and read or written to ESRI ASCII files or to GeoTiff via GDAL.

# Read a grid and resample
grid = karta.read_gtiff("LC80360272015200.tif").resize(bbox)
grid = grid.resample(90, 90)

Bounding boxes, distances and lengths, intersection calculations, convex hulls

Karta provides Point, Line, Polygon, and Multipoint classes, and computes properties and relationships vector geometries.

Point, Multipoint, Line and Polygon are all subclasses of Geometry. All geometry objects can contain singlepart and multipart metadata, as well as a reference to a coordinate reference system object. Methods of geometry objects handle basic queries about geometry properties and relationships.

GeoJSON I/O is handled natively while ESRI shapefile access is via GDAL/OGR. Geometries implement the __geo_interface__ attribute, permitting interchange with other libraries such as Shapely.

Placing data on Earth with coordinate reference systems

In a given project, GPS measurements may be in WGS84 spherical coordinates, while survey is on a UTM grid and public roads data are in a local Albers projection. Karta is coordinate-system aware and tracks the reference system associated with grids and geometries.

Geometries and Grids are referenced to coordinate reference system objects (karta.crs.CRS) that provide information about the geographical projection and geodetic relationships between positions.

For performing accurate calculations, geodetic operations rely on the algorithms from geographiclib, while projections leverage pyproj.
